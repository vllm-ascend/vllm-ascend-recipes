# vLLM-Ascend Recipes

Deploy any model on Ascend NPUs — community-maintained vllm-ascend deployment recipes.

Pick a model, choose your hardware, copy the `vllm serve` command and run. Covers Atlas 800 A2/A3 and other Ascend NPU hardware.

## Local Development

```bash
pnpm install
pnpm dev
```

Visit `http://localhost:4321/vllm-ascend-recipes/`.

## Build

```bash
pnpm build
```

Output in `dist/`.

## Directory Structure

```
models/
├── en/          # English content (maintained by developers)
│   ├── Qwen/
│   └── deepseek-ai/
└── zh/          # Chinese content (generated by translation tools)
    ├── Qwen/
    └── deepseek-ai/
```

## Adding a New Model

1. Create `models/en/{provider}/{model-name}.yaml`
2. Fill in model info following the schema (see existing YAML for reference)
3. Validate: `pnpm validate`
4. Preview: `pnpm dev`
5. Generate Chinese translation in `models/zh/{provider}/{model-name}.yaml`

### YAML Schema

| Field | Required | Description |
|-------|----------|-------------|
| `hf_id` | Yes | HuggingFace model ID |
| `title` | Yes | Display name |
| `provider` | Yes | Provider name |
| `description` | Yes | Short description |
| `architecture` | Yes | `dense` or `moe` |
| `parameters` | Yes | Parameter count |
| `active_parameters` | MoE only | Activated parameters |
| `context_length` | Yes | Native context length |
| `modality` | Yes | text/multimodal/audio/image |
| `updated` | Yes | Update date |
| `overview` | Yes | Model overview |
| `weight_download` | Yes | Weight download (multi-version, multi-source) |
| `quantization` | No | Quantization guide |
| `prerequisites` | No | Prerequisites |
| `env_setup` | Yes | Environment setup (pip and/or container) |
| `scenarios` | Yes | Deployment scenarios (cascade: NPU → precision → deployment mode) |
| `performance_accuracy` | No | Accuracy results |
| `performance_benchmark` | No | Benchmark results |
| `references` | Yes | Reference links |

## i18n

- Default language: English
- Language toggle: EN / 中文 in the header
- Developers maintain `models/en/`, translation tools generate `models/zh/`
- If `models/zh/` is missing a model, it falls back to English

## Deployment

Push to `main` branch — GitHub Actions auto-builds and deploys to GitHub Pages.

Configure in repo Settings → Pages → Source: GitHub Actions.

### PR Preview

When a PR targets `main`, CI builds the static site and uploads it as a workflow artifact, then comments download/preview instructions on the PR. Reviewers download the artifact, unzip, and serve locally (e.g. `npx serve .`) to inspect the rendered site before merge. When the PR is closed/merged, the comment is marked outdated; artifacts expire automatically after 14 days.

> **Note:** PRs from forks cannot post comments or upload artifacts with the default `GITHUB_TOKEN`. Fork contributors may need a maintainer to run the workflow manually.
